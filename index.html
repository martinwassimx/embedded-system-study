Chapter 4

Pipelining

Do more work at once — like a smart laundry day.

🧺
The Laundry Idea

Wash (30 min) → Dry (40 min) → Fold (20 min). Doing 4 loads one-at-a-time takes 6 hours. If you start the next load as soon as a machine is free (pipelining), it takes only 3.5 hours — same work, way less waiting!

The 5 Stages of MIPS

Every instruction flows through five stages, one per clock cycle. While one instruction is being executed, the next is already being decoded — that's the magic.

Instr 1IFIDEXMEMWB
Instr 2IFIDEXMEMWB
Instr 3IFIDEXMEMWB
📥

IF · Instruction Fetch

Get the instruction from memory; PC ← PC + 4.

🔍

ID · Decode / Reg Fetch

Decode it and read the source registers (A & B).

🧮

EX · Execute

ALU does the math, or computes a memory address.

💾

MEM · Memory

Load or store data (ALU ops do nothing here).

✍️

WB · Write Back

Write the result back into the register file.

💡
Key lesson

Pipelining doesn't speed up a single task's latency — it boosts throughput of the whole workload. Speed is limited by the slowest stage, and filling/draining the pipe costs a little.

The 3 Hazards ⚠️

Hazards are conditions that cause wrong behavior if not handled. Common fix: stall the pipeline (insert a “bubble” / NOP).

🏗️

Structural

Two instructions need the same hardware at once (e.g. one memory for fetch + data).
Fix: duplicate hardware / Harvard architecture (separate I & D memory).

🧦

Data

An instruction needs a result not ready yet (the “missing sock”).
Fix: stall (bubble) or forwarding/bypassing — send the result straight to the next instruction.

🔀

Control

A branch changes which instruction comes next (BEQ R1,R2,LABEL).
Fix: stall, predict “not taken,” or use branch predictors. Penalty ≈ 2–3 cycles.

🔢
Branch math example

20% branches, penalty 3 cycles → CPI = 1 + (0.2 × 3) = 1.6. That's a 60% performance drop — so branch prediction really matters!

Chapter 6

The Pareto Principle (80/20)

A few causes drive most of the results.

🎯
The Rule in one line

80% of outcomes come from 20% of causes. The numbers aren't exact — they describe an imbalance. Named after economist Vilfredo Pareto, who saw 80% of Italy's land owned by 20% of people; later generalized by Joseph Juran for quality.

🔥 Vital Few (20%)

Cause most of the impact. High importance → focus your resources here.

Prioritize

🌊 Trivial Many (80%)

Contribute little impact. Lower priority — but don't ignore them completely.

Everyday Examples

💰

Business

20% of products generate 80% of revenue.

🐛

Software

20% of modules cause 80% of bugs.

📚

Education

20% of material appears in 80% of exams.

⏰

Time

2 productive hours give 80% of results.

🏭

Manufacturing

20% of defect types cause 80% of complaints.

📈

Pareto Chart

Bars (sorted high→low) + cumulative % line, with an 80% threshold.

Worked Example — 100 Defects

Sort defects, find each %, add cumulative %, then spot where you cross 80%.

Defect A
50 · cum 50%
Defect B
30 · cum 80%
Defect C
10 · cum 90%
Defect D
6 · cum 96%
Defect E
4 · cum 100%
✅
Conclusion

Defects A + B = 80% of all problems (the vital few). Fix those two first and you eliminate most defects — saving time, cost, and effort.

Chapter 7

Interrupts

“Tell me when you need me” beats “are we there yet?”

Polling vs. Interrupt

🔁 Polling

CPU constantly checks a device's status in a loop. Ties down the CPU and wastes cycles.

while(true){
  if(PIND.2==0)
    // do something;
}

⚡ Interrupt

Device signals the CPU only when needed. Efficient, has priority, and can be masked.

main(){
  // common task
}
// when PIND.2==0
//   do something
More efficient

How an interrupt is serviced

1 · Save PCStop fetching, save where you were.
2 · Find ISRLook up address in the IVT.
3 · Run ISRExecute the handler function.
4 · ResumeRestore PC, continue as before.
ISR
Interrupt Service Routine — the function that runs when an interrupt fires.
IVT
Interrupt Vector Table — stores the memory address of each ISR.
Vector No.
Lower number = higher priority. INT0 > INT1.
SREG bit 7 (I)
Global interrupt enable. sei() sets it; disabled on reset.
🔌
Interrupt sources & triggers

Sources: external switches, timers, serial port, PWM, ADC. External INT0/INT1 can be edge-triggered (on a signal change) or level-triggered (on a specific value, low by default).

5 Steps to Program an Interrupt

1

Include the header

#include <avr/interrupt.h>

2

Define the handler

Use the ISR(vector_name) macro (updates the IVT).

3

Enable the specific interrupt

e.g. GICR |= (1<<INT0);

4

Configure trigger details

Set MCUCR for edge/level (e.g. falling edge).

5

Enable globally

Call sei(); to switch on the interrupt subsystem.

💻 Example: toggle PORTB.5 when INT0 goes low 
#include <avr/io.h>
#include <avr/interrupt.h>
int main(){
  DDRB = 1<<5;        // PB5 output
  MCUCR = 0x2;        // INT0 falling-edge
  GICR |= (1<<INT0);  // enable external INT0
  sei();              // enable interrupts
  while(1);           // wait here
}
ISR(INT0_vect){       // handler
  PORTB ^= (1<<5);    // toggle PB5
}

Chapter 8

Evaluating System Performance

Be correct first — then be fast.

🎯
Goals

Ensure correctness → improve speed → optimize resource usage → enhance user experience. Correctness always comes first.

Performance Metrics

✅

Correctness

Accurate results (functional + logical).

⚡

Speed

Execution time / latency.

📦

Throughput

Tasks completed per unit time.

💻

Resource Use

CPU, memory, disk usage.

📈

Scalability

Ability to handle growth.

🔍

Profiling

Analyze behavior to find bottlenecks.

Profiling Tools

gprof
Time per function, call counts, and who-calls-whom → find hotspots.
perf
Low-level Linux tool: CPU cycles, cache hits/misses, branch mispredicts.
VisualVM
GUI profiler for Java: memory, CPU, threads, leaks.
Intel VTune
Deep hardware profiling for C/C++ & parallel apps.

Compiler Optimizations

🔁

Loop Unrolling

Do several iterations per pass to cut loop-control overhead.

📥

Inline Expansion

Replace a function call with its body — no call overhead.

🗑️

Dead Code Elimination

Remove unreachable code or unused results.

🧮

Constant Folding

Evaluate constant expressions at compile time.

📐
Pareto meets optimization

80% of performance issues come from 20% of the code. Profile → find hotspots → optimize critical sections → re-measure. Don't waste time on low-impact areas.

⚡ Chapter 4 — numbers & terms worth memorizing

4500 ns
Single-cycle (45ns × 100)
4600 ns
Multicycle (10ns × 4.6 × 100)

Pipelined (10ns × 104)
CPI 1.6
20% branches × 3-cycle penalty
Throughput vs. Latency
Pipelining raises throughput (instructions finished per second) but does not shorten a single instruction's latency.
Speedup
Ideal speedup ≈ number of stages. Reduced by unbalanced stages and fill/drain time.
Forwarding (Bypassing)
Send an ALU result straight to a waiting instruction to avoid a stall.
Harvard Architecture
Separate instruction & data memories — removes the structural hazard of one shared memory.
Squash / Flush
Discard wrongly-fetched instructions after a taken branch.
CPI Formula
CPI = 1 + (branch frequency × penalty). Here 1 + 0.2×3 = 1.6 → ~60% slower.

📊 Chapter 6 — Pareto cheat sheet

🧮
The 100-defect example, fully worked

Counts: A=50, B=30, C=10, D=6, E=4. Cumulative %: A=50 → B=80 → C=90 → D=96 → E=100. Vital few = A + B (80%); trivial many = C, D, E (20%). Fix A & B first to remove ~80% of all defects.

Origin
Vilfredo Pareto (Italian economist): 80% of Italy's land owned by 20% of people. Generalized by Joseph Juran for quality.
Pareto Chart
Bars sorted high→low + a cumulative-% line + an 80% threshold line.
Strength
Simple, fast, great for prioritizing the highest-impact causes.
Limitation
Not mathematically exact — a guideline, not a rule. Don't ignore the other 80% entirely.

🔔 Chapter 7 — AVR interrupt quick reference

Polling vs Interrupt
Polling ties up the CPU checking devices; interrupts let the device signal only when it needs service.
ISR / IVT
ISR = the handler routine. IVT = table holding each ISR's address.
Priority
Lower vector number = higher priority, so INT0 outranks INT1.
Global Enable
Bit 7 (I) of SREG; set with sei(). Cleared on reset and auto-cleared inside an ISR.
Key Registers
MCUCR sets edge/level trigger; GICR enables INT0/INT1. INT0=PD2, INT1=PD3.
5 Steps
Include header → write ISR() → enable specific interrupt → configure trigger → sei().
💡
Toggle PB5 on INT0 (Example 2)

DDRB=1<<5; MCUCR=0x2; GICR|=(1<<INT0); sei(); while(1); with ISR(INT0_vect){ PORTB^=(1<<5); } — MCUCR=0x2 selects falling-edge trigger.
